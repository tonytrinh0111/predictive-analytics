The R Environment

Base R

Base R can be viewed as the “engine” behind all of the calculations that R performs. You can fully operate R calculations with just the base R without RStudio. Base R can be downloaded from https://cran.r-project.org/. You will only see the R console and R script file when operating in base R. There is no visualization of the workspace.

RStudio

RStudio is the “front-end” or the “user interface” that allows users to code and operate in R more easily. Not only that you can view the script and the console, you can also visualize the workspace environment, the working directory, a collection of plots that have been generated in the current session, the available and loaded packages, amongst other things.

RStudio can be downloaded from https://www.rstudio.com/. For the purpose of the Master of Business Analytics courses, the free desktop version is adequate. Commercial licenses and RStudio server services are also available for a fee.

R Packages

R was developed as an open source statistical analysis software (as an alternative to SPlus). A very attractive feature of R is its collegiate nature, where packages can be developed and shared on the R repository. Anyone who has access to R has access to these contributed packages. Many researchers develop an R package as part of their research output, with an aim to make their work more accessible.

There is also a drawback to this feature. Because R and associated packages are open source, anyone can make a contribution to the repository. Thus, the users need to have a certain level of expertise in statistical analysis to judge the credibility of the packages. Always check who is the contributor of the package, and check whether there are refereed research outputs associated with the models/analysis that the package is intended for. Your own statistical knowledge and research skills will also help in making these judgements.

Installing a package is a matter of calling a simple command, given that you have internet connection. For example, to install the “fpp3” package, you simply call:

#Installing a package - you only need to do this once
install.packages("fpp3")

After a package has been installed, you will need to call in the “library” if you wish to use the functions from the packages. You do not need to install the package everytime.

#Telling R that you wish to use this package for the current session.
#You need to do this every time a new session is started.
library("fpp3")

Note that there will be times when function names are duplicated in multiple packages. A warning message will show if this is the case, and the function will be overwritten by the function of the most recent library that you call in.

R as an Object Oriented Software

R is an object oriented software. An assignment of a certain object to a name is stored in the global environment. You may elect to save all of these objects in a workspace using the file extension “.RData”. If you do not save your workspace, all objects are deleted when R is shut down.

There are many object types that R can handle. Most of the time you will be dealing with arrays, matrices and data frames objects. If you use a certain package to estimate a model, it will generate a “model fit” object, which is basically a collection of objects that record the results. This is package specific so knowing how to read the package documentation is very important.

Basic Syntax and Operations

R is an expression language with simple syntax. You should take note of the following items:

  • Comments are preceeded by “#”.
  • R is case sensitive, so that an object named “abc” is different to an object named “ABC.”
#Case sensitivity
abc="abc"
ABC="ABC"

abc
## [1] "abc"
ABC
## [1] "ABC"
  • The two fundamental types of syntax are assignment and expression. If your code only consist of an expression, R will evaluate the operations and print the results. Nothing is saved. If you assign the expression to an object, then the results are saved in that particular object.
  • Assignment can be done with “=” sign, or more traditionally “<-”.
#Examples of expressions
4*5
## [1] 20
5*(6+7)
## [1] 65
#Examples of assignment - "=" and "<-" do the same job

y<-5

#To look at what x and y contain:

## [1] 4
y
## [1] 5
#Examples of assignment of expression
a<-4*5
a
## [1] 20
b<-5*(6+7)
b
## [1] 65
c=a+b

## [1] 85

Useful Object Types

Vectors, Arrays, Matrices and Data Frame

R is considered to be a matrix language, and as mentioned before, you will be dealing with vectors, arrays, matrices and data frames objects in most applications.

  • A vector is a collection of cells with fixed size (length). It only has one dimension.
  • A matrix is a collection of cells with fixed length (rows) and width (columns). That is, a matrix has two dimensions.
  • An array is a collection of cells with with fixed length (rows) and width (columns), but can have “n” dimensions. If n=1, the array is equivalent to a vector, and if n=2, the array is equivalent to a matrix.
# A 3x1 vector of zeros
v=c(0,0,0)
# Checking the length of v
length(v)
## [1] 3
# A 3x2 matrix, filled with zero
m=matrix(0,nrow=3,ncol=2)
# Checking the dimension of m
dim(m)
## [1] 3 2
# An array of size 3x2x2, filled with zero
a=array(0,c(3,2,2))
#checking the dimension of a
dim(a)
## [1] 3 2 2

In data modelling, we often deal with a table of data, where each column represents a certain variable, and we may have a collection of these variables. Such a table consists of a certain number of observations (row), and a certain number of variables (column). This can be stored in a matrix since it has two dimension, but most packages in R actually prefer to deal with “dataframes”. The object type “dataframe” is essentially a table of data, where each cell in a particular column is of the same type and a column name is allowed.

#An example of creation of data frame. 
#Name is a collection of characters
name=c("Abby","Bob","Chris")
#Height is numeric
height=c(165,188,179)
#A dataframe can contain data of different types
data=data.frame(name,height)
#Naming the variables makes it easier to access later
names(data)=c("name","height")

Model Fit Object

Once you estimate a statistical model and assign a result to a certain name, you will be able to access a number of items the estimation function generates. This section will give an example of a linear regression model which is estimated using the “lm()” function and is available as part of base R (you do not need to call in an additional package for this).

#Let's take an example data from "fpp" package
library(fpp)
data(fuel)

#Fit a model of Carbon=b0+b1*City+error
fit=lm(Carbon~City,data=fuel)

The object “fit” is assigned the model fit object that is a result of the linear model (lm) fit. If you look into the Global Environment after running this command line, “fit” is a “List of 12”, which means that it is a collection of 12 objects. To look at what the list contains, you can click on the arrow next to the object in the Global Environment, or use the View(fit) command. You can call out the component as needed. For example:

#Taking a look at the coefficients

Setting the Working Directory

If your analysis involve importing the data file from a certain file directory from your computer, you need to set the working directory to that particular location. You can do this by executing the following code. You can alter the directory location by changing the text within the quotation mark to your own specific location.

#Setting work directory to the location specified in the quotation marks
setwd("C:\Users\Ole\Document\Predictive")

If you wish to set your working directory to the location where your code file is currently saved, you can do so using the RStudio user interface. Go to Session > Set Working Directory > To Source File Location.

More Advanced Programming in R

R is like any other programming language, where you can construct the code for your own purpose. These may include writing your own functions, so that if you would like to repeat certain calculations you can do so with ease. If you require multiple calculations to be conducted, you may also choose to write a program loop around your calculations.

Writing Your Own Functions

If you need to run certain lines of command multiple times, it is most efficient to write a function and call the function instead of writing the code out explicitly. The key ingredients to a function are:

  • The inputs that can change everytime you evaluate a function
  • The expression(s) to be executed with the inputs
  • The outputs that varies depending on the input

Let’s have a look at a simple function to compute relative change in two values and return the relative change in 1) percentage and 2) decimal.

#A function to compute relative change from x1 to x2, where x1 is the base value
relative_change<-function(x1,x2){
  dec=(x2-x1)/x1
  perc=dec*100
  return(list(percentage=perc,decimal=dec))
}

#Let's call the function
result=relative_change(100,101)
result$percentage
## [1] 1
result$decimal
## [1] 0.01

Make sure that the function is named appropriately, so that the user can roughly get an idea of what the function does. The function should also be well documented for future uses.

IF/FOR/WHILE Loops

Just like any other programming language, you can also construct “if/else” conditions, for and while loops. The purposes of these commands are:

  • if/else statements - for execution of expressions when the condition of the “if” is met. If the condition is not met, then construct the expressions given under the “else” command. Note that when no action is required under the else command, there is no need to define it.
#Compute the relative change if x1<x2, do nothing otherwise.
x1=100
x2=101

if(x1<x2){
  rc1=relative_change(x1,x2)
}

  • The “for” loop is for when you would like to execute the expressions in it multiple times. Here is an example:
#This is a loop to compute the relative change multiple times, 
#depending on the size of x1 and x2 (they must be of the same size)

#seq() is a command to generage a sequence
x1=seq(from=100,to=120,by=1) 
x2=seq(from=101,to=121,by=1)

#first find out the length of x1
n=length(x1)
#create storate vector for percentage and decimal changes
perc=matrix(0,n,1)
dec=matrix(0,n,1)
for(i in 1:n){
  #store the list in a tmp object
  tmp=relative_change(x1[i],x2[i])
  #allocate percentage and decimal changes appropriately
  perc[i]=tmp$percentage
  dec[i]=tmp$decimal
}

  • The while loop repeats the expression within it until the condition within the bracket is violated.
#Let's use x1 and x2 generated by the above for loop example. 
#Construct a while loop to compute the relative change while the change>0.9%.
#Record the relative changes that are >=0.9%.

#set a tmp value for the condition
chg=1
#set an index for access to x1 and x2
i=1
while(chg>0.9){
  tmp=relative_change(x1[i],x2[i])
  if(i==1){
    #if this is the first computation, assign a value
    perc2=tmp$percentage
    dec2=tmp$decimal
  }else if(tmp$percentage>0.9){
    #otherwise, add the new results to existing ones.
    #"rbind" is used to add additional rows.
    #The second layer of "if" is there to trap out 
    #the final result that may be less than 0.9%.
    perc2=rbind(perc2,tmp$percentage)
    dec2=rbind(dec2,tmp$decimal)
  }
  #reset the condition
  chg=tmp$percentage
  #increment the index to move on to the next element
  i=i+1
}

Your Journey with R

In the subject Predictive Analytics, we aim to use R to explore the various tools that allow us to do predictions. The general aim is to use R as a tool to illustrate the use of predictive modelling and conduct predictive evaluations. You are not expected to become an expert in R computations, but are expected to be able to make sense of the outputs.

If you wish to develop expertise in R, keep in mind that your skills will grow with hands-on experience and time. Do not expect to become an expert in R in a short period of time. It is also worth to note that R syntax and output formats can be package specific, so there is no single person who is expert in every R packages. The key to using R is to be familiar with base R operations, and be resourceful when it comes to usage of contributed packages.

Here are some tips that you may find useful:

  • Make use of online resources and forums. Here are some useful spots:

    1. https://www.r-bloggers.com/
    2. https://blog.rstudio.com/
    3. https://stackoverflow.com/questions/tagged/r
    4. https://stats.stackexchange.com/
    5. https://rstudio.com/resources/cheatsheets/

Remember that these blogs/forums are open to the public, so use your discretion!

  • When looking into packages, look at the documentation of the package, academic references, credibility of the authors, and search for any criticisms in the above resources. The Journal of Statistical Software (https://www.jstatsoft.org/) also provide proper backgrounds and some validation of the packages.

  • Use the Help function!

  • Read and make sense of example codes that are given (or ones that you find in package documentations/online) carefully. Make sure you understand what they are doing.

  • Once you have some idea of the structure of the code, try to replicate the modelling exercise yourself to consolidate the materials. This is what the syndicate tasks are designed for. The more exposure you have to the analysis and R (and the relevant packages), the faster you will learn, so get your hands dirty!

  • If you have any questions at all about class codes, be sure to shout out your questions to clarify your questions straightaway.

  • Finally, practice makes perfect, and it even makes the experience more enjoyable if you have fun!
